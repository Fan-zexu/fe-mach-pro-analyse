# 第二章：编译时优化系统

> 一句话概括：MachPro 的 babel 插件将 JSX 在编译阶段转为直接 DOM 操作指令（insert/attr/style/listen），跳过运行时 VDOM diff，灵感源于 SolidJS 但针对 Native 渲染做了深度适配。

## 2.1 编译策略对比

| 框架 | JSX 编译目标 | 运行时行为 |
|------|------------|-----------|
| React | `React.createElement()` → VDOM 节点 | 运行时 diff VDOM 树，计算最小 DOM 操作 |
| Preact | `h()` → VDOM 节点 | 运行时 diff，但更轻量 |
| SolidJS | 直接 DOM 操作指令 | 无 VDOM，响应式更新 |
| **MachPro** | **`insert`/`attr`/`style`/`listen` 指令** | **无 VDOM diff，直接操作模拟 DOM** |

## 2.2 编译产物分析

虽然 `babel-plugin-mach-pro` 不在当前代码库中，但我们可以从 `mach-pro-core/src/dom/dom.ts` 的函数签名和 `process/fragment.ts` 的结构**反推编译器的输入输出**。

### 编译前（业务代码）

```jsx
function UserCard({ name, avatar, onClick }) {
  const [count, setCount] = useState(0);
  
  return (
    <view style={{ padding: 10 }}>
      <image src={avatar} mode="cover" />
      <text style={{ fontSize: 16 }}>{name}</text>
      <view onClick={() => setCount(count + 1)}>
        <text>点击次数: {count}</text>
      </view>
    </view>
  );
}
```

### 编译后（推测产物）

```javascript
function UserCard(props, $$) {
  const [count, setCount] = useState(0);
  
  // Fragment: 静态模板定义 + 动态部分绑定
  return f({
    key: 'UserCard_frag',
    $$: { scope: { count, setCount, props } },
    
    // d: define -- 静态 DOM 模板（只在首次执行）
    d: (ctx) => {
      const el = t({                    // temp: 创建静态 DOM 树
        t: 'view',                      // tagName
        c: [                            // children
          { t: 'image' },
          { t: 'text' },
          { t: 'view', c: [{ t: 'text' }] }
        ]
      });
      // child: 通过相对位置获取子节点引用并缓存
      const img = c('c1', el, 'firstChild');
      const txt = c('c2', img, 'nextSibling');
      const btn = c('c3', txt, 'nextSibling');
      const btnTxt = c('c4', btn, 'firstChild');
      return [el, img, txt, btn, btnTxt];
    },
    
    // r: render -- 动态部分绑定（每次更新都执行）
    r: (ctx, $$) => {
      const { count, setCount, props } = $$.scope;
      const [el, img, txt, btn, btnTxt] = ctx.$d;
      
      s(el, 'padding', 10);              // style: 设置样式
      a(img, 'src', props.avatar);        // attr: 设置属性
      a(img, 'mode', 'cover');
      s(txt, 'fontSize', 16);
      i('i1', txt, props.name);           // insert: 插入动态文本
      l('l1', btn, 'click', () => setCount(count + 1));  // listen: 事件绑定
      i('i2', btnTxt, '点击次数: ' + count);
      
      return el;
    }
  });
}
```

### 单字母别名映射表

| 别名 | 全名 | 功能 | 源码位置 |
|------|------|------|---------|
| `i` | `insert` | 插入动态内容到父节点 | dom.ts L81-L218 |
| `l` | `listen` | 事件监听绑定 | dom.ts L220-L238 |
| `a` | `attr` | 属性设置 | dom.ts L242-L301 |
| `s` | `style` | 样式设置 | dom.ts L332-L362 |
| `t` | `temp` | 从模板描述创建 DOM 树 | dom.ts L366-L381 |
| `c` | `child` | 通过相对位置获取子节点 | dom.ts L387-L395 |
| `f` | `fragment` | Fragment 上下文工厂 | process/fragment.ts |

单字母别名的设计目的是**压缩编译产物体积**——在高频调用的场景下（一个页面可能有数百个 insert/attr 调用），每个函数名省 5-6 个字符，累计可减少可观的 bundle size。

## 2.3 三种核心元素类型

编译器将 JSX 转换为三种运行时表示：

### Fragment（静态模板片段）

Fragment 对应**静态的 JSX 结构**。编译器识别出不含动态表达式的 JSX 子树，将其提取为 `temp()` 调用创建的 DOM 模板。

关键特征：**只在首次创建时执行 `define`（创建 DOM 节点），后续更新只执行 `render`（绑定动态值）**。

```
首次渲染: define(ctx) → 创建 DOM 模板 → render(ctx) → 绑定动态值
后续更新: render(ctx) → 只更新变化的动态值
```

源码位置：`mach-pro-core/src/react/process/fragment.ts`

### Block（代码块/条件渲染）

Block 对应**动态的渲染输出**。每个组件的 `render()` 结果都被包装为一个 Block。条件渲染（`{condition && <Comp />}`）也会生成 Block。

关键特征：**mount 时递归执行 body 函数直到非函数值，update 时重新执行并 diff**。

源码位置：`mach-pro-core/src/react/process/block.ts`

### Component（组件节点）

Component 对应 React 组件。通过 `initComponent` 区分类组件和函数组件，创建实例后调用 `$init` 挂载。

源码位置：`mach-pro-core/src/react/process/component.ts`

## 2.4 编译时 vs 运行时的职责划分

| 编译时完成 | 运行时完成 |
|-----------|-----------|
| JSX → DOM 操作指令 | 执行 DOM 操作指令 |
| 静态模板提取为 Fragment | Fragment 的 define (首次) 和 render (更新) |
| 动态表达式标记为 insert/attr/style | insert 的 marker+cache 增量更新 |
| 事件处理函数提取为 listen 调用 | listen 的 Map 式事件替换 |
| 环境变量替换（MACH_ENV）→ 死代码消除 | 组件生命周期管理 |
| 单字母别名替换 | Hooks 状态管理 |

### 性能收益

1. **跳过 VDOM diff**：传统 React 需要创建 VDOM 树 → diff 新旧 VDOM → 计算最小 DOM 操作。MachPro 编译后直接执行 DOM 操作，省去了 VDOM 创建和 diff 的开销。
2. **静态模板只创建一次**：Fragment 的 `define` 只在首次执行，后续更新只调用 `render` 绑定变化的动态值。
3. **属性/样式多层短路**：`attr` 函数有 4 层比较（引用→浅比较→空值→number/string），`style` 函数有 `$styleList` 缓存，大部分无变化的属性在第一层就短路返回。

## 本章小结

MachPro 的编译时优化是整个框架最核心的技术创新。通过 babel 插件将 JSX 在编译阶段转为 insert/attr/style/listen 等直接 DOM 操作指令，运行时跳过 VDOM diff。Fragment 静态模板实现"只创建一次，只更新变化"，Block 处理条件/动态渲染。单字母别名压缩产物体积。这套策略类似 SolidJS，但针对 Native 渲染场景做了深度适配。

## 面试素材

### 高频面试题

**基础题**：MachPro 的编译时优化具体做了什么？与传统 React 有什么区别？

**深度题**：Fragment 的 define/render 分离设计解决了什么问题？请画出一个组件从 JSX 到屏幕显示的完整链路。

### 参考回答

> MachPro 的编译器将 JSX 转为直接 DOM 操作指令，跳过了传统 React 的 VDOM 创建和 diff 过程。具体来说，静态 JSX 结构被提取为 Fragment 的 `temp()` 模板（只创建一次），动态部分被转为 `insert`/`attr`/`style`/`listen` 调用（每次更新只处理变化的部分）。与 SolidJS 的编译策略类似，但 MachPro 还针对 Native 端做了适配，比如样式单位自动转换、gradient 属性映射、image mode 兼容等。

### 亮点话术

> "在编译策略的选型上，我们参考了 SolidJS 的思路——将 JSX 在编译阶段拆分为'静态模板'和'动态绑定'两部分。但与 SolidJS 不同的是，我们面向的是 Native 渲染环境，所以 temp 函数创建的不是真正的浏览器 DOM，而是 mach-pro-render 模拟的 Element 实例，底层通过 super 调用驱动客户端原生渲染引擎。这种设计让我们在保持编译时优化收益的同时，无缝适配了 Native 的渲染管线。"
