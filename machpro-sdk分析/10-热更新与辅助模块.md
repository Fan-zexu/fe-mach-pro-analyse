# 第十章：热更新与辅助模块

> 一句话概括：MachPro 的热更新通过 CDN 版本管理实现 bundle 增量更新，辅助模块覆盖 KNB JSBridge、H5 降级、S3 上传和 Native 调试等能力。

## 10.1 热更新模块（mach-pro-update）

### 更新机制

MachPro 页面以 bundle 为单位部署到 CDN，客户端在页面打开时检查版本：

```
客户端缓存 bundle v1.0
    ↓ 打开页面
检查 CDN 最新版本
    ↓ 发现 v1.1
下载 diff patch / 全量 bundle
    ↓
应用更新，刷新页面
```

### 版本管理

通过 Lerna 统一管理版本号（当前 v3.6.2-next.61），`scripts/sync-version.js` 同步版本到各 example：

```javascript
// scripts/sync-version.js
// 将 lerna.json 中的版本号同步到 examples 的 package.json
const version = require('../lerna.json').version;
// 遍历 examples 目录更新依赖版本
```

### 发布链路

```
lerna publish → npm registry (r.npm.sankuai.com) → Talos 平台部署 → CDN 分发
```

支持四种发布通道：

| 通道 | 命令 | tag | 用途 |
|------|------|-----|------|
| 正式版 | `npm run release` | latest | 线上稳定版 |
| Beta 版 | `npm run release:beta` | beta | 灰度测试 |
| Alpha 版 | `npm run release:alpha` | alpha | 内部测试 |
| Next 版 | `npm run release:next` | next | 预发布 |

## 10.2 KNB JSBridge 封装

### mach-pro-knb

封装美团客户端的 KNB（Kernel Native Bridge）接口，提供统一的 JS 调用 Native 能力的 API：

```javascript
import KNB from 'mach-pro-knb';

// 统一接口，屏蔽 iOS/Android 差异
KNB.invoke('dialog', {
    title: '提示',
    content: '确认操作？',
    buttons: ['取消', '确认']
}, (res) => {
    if (res.buttonIndex === 1) { /* 确认 */ }
});
```

### mach-pro-knb-adaptor

KNB 接口适配层，处理不同客户端版本的 KNB API 差异。

## 10.3 H5 降级体系

### mach-pro-h5-element

通过 Web Components 在 H5 环境中模拟 Native 自定义标签：

```javascript
// 注册 Custom Element
customElements.define('scroll-view', class extends HTMLElement {
    connectedCallback() {
        this.style.overflow = 'auto';
        // 模拟 Native scroller 的行为
    }
});

customElements.define('mt-image', class extends HTMLElement {
    static observedAttributes = ['src', 'mode'];
    attributeChangedCallback(name, oldVal, newVal) {
        if (name === 'src') this._loadImage(newVal);
    }
});
```

### mach-pro-h5-native-module

H5 端 Native Module 模拟层，将 `Mach.requireModule` 调用映射到 Web API：

```javascript
const h5Modules = {
    WMNetwork: {
        request: (params) => fetch(params.url, { ... }),
    },
    WMRouter: {
        navigateTo: (url) => window.location.href = url,
        navigateBack: () => window.history.back(),
    },
    WMStorage: {
        getItem: (key) => localStorage.getItem(key),
        setItem: (key, value) => localStorage.setItem(key, value),
    },
};
```

## 10.4 静态资源上传（mach-pro-s3）

封装 MOS S3（美团对象存储）的上传接口，在构建流程中自动将图片等静态资源上传到 CDN：

```
构建产物 → mach-pro-s3 上传 → CDN URL → 注入到 bundle 中
```

## 10.5 Native 调试工具（mach-pro-native-devtools）

在 Native 环境中提供类 Chrome DevTools 的调试能力：

功能包括组件树可视化（类似 React DevTools）、Store 状态查看、性能面板（FPS/内存/DOM 节点数）、日志控制台和网络请求监控。

调试工具通过 `rollup-plugin-injectjs` 在开发环境构建时注入，生产环境通过 tree-shaking 移除。

## 10.6 跨端组件库（mach-pro-components）

基于 mach-pro-core + mach-pro-store 构建的跨端组件库，提供常用 UI 组件的三端统一实现。通过 `MACH_ENV` 条件编译处理平台差异。

## 10.7 微信小程序标准组件（wechat-standard-components）

小程序端专用的标准组件实现，对齐微信小程序原生组件的 API 和行为。

## 10.8 类型定义（types）

全局 TypeScript 类型定义，包括 Mach 全局对象、Native Module 接口、DOM API 扩展等类型声明。

## 本章小结

MachPro 的辅助模块围绕三个方向展开：1) 热更新和版本管理通过 Lerna + CDN + Talos 实现 bundle 级增量发布；2) H5 降级体系通过 Web Components + Web API 映射实现 Native 标签和 Module 的 H5 回退；3) 开发工具链覆盖 KNB 封装、S3 上传、Native DevTools、QJsc 预编译等。这些模块共同构成了 MachPro 从开发到部署的完整工程体系。

## 面试素材

### 高频面试题

**基础题**：MachPro 的 H5 降级是怎么做的？

### 亮点话术

> "H5 降级是分两层做的。标签层面，通过 Web Components 注册 Custom Elements 模拟 Native 自定义标签（如 scroll-view、mt-image），保证 JSX 渲染不报错。API 层面，通过 h5-native-module 将 Mach.requireModule 调用映射到对应的 Web API（WMNetwork→fetch，WMRouter→location，WMStorage→localStorage）。整体设计保证同一份业务代码在 H5 环境中也能正常运行，只是功能可能有所降级。"
